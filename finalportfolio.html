Major Project 1

Technical Document Analysis

Context

This project involved selecting a piece of existing technical documentation and analyzing its effectiveness, audience appropriateness, and accessibility compliance. The goal was to identify gaps in the current communication strategy.

Rhetorical Strategy

I utilized the deficit model of communication as a counter-example to demonstrate how the original document failed to meet user needs. My revision strategy focused on plain language principles and structural hierarchy to improve readability for non-expert users.

Major Project 2

Instructional Guide Proposal

Context

For the second major project, I proposed a new set of instructional materials for a complex software tool used by introductory CS students. The project required audience analysis and a formal proposal format.

Rhetorical Strategy

The strategy relied heavily on ethos and logos. I established credibility by referencing common pain points validated by student surveys (logos) and proposed a solution that aligned with the course's pedagogical goals.

Major Project 3

Accessibility as Foundation

Context

Moving beyond "retrofitting," this project centers accessibility as a foundational part of the design process. It anticipates user needs from the outset rather than waiting for complaints to signal access barriers.

Rhetorical Strategy

Drawing on Lisa Pappas's Rhetorical Accessability, I argue that technical communicators have an ethical responsibility to consider delivery means. The artifact demonstrates this through proper heading structures, meaningful alt text for data visualizations, and accessible data tables.
